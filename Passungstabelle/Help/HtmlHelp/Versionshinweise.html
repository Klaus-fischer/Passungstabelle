Versionshinweise

Version 8.2.0.2

       Es wird auf freistehende Bemaßungen geprüft

       Freistehende Bemaßungen und Bemaßungen bei denen der Maßwert 0 ist werden übersprungen

       d.h. Das Add-In ist nicht auf abgelöste Zeichnungen anwendbar

Version 8.2.0.1

       Bei linearen Bemaßungen, denen ein Ø Zeichen mit zusätzlichem Text vor dem Ø Zeichen angefügt wurde,

       wurde das Ø Zeichen in der Passungstabelle nicht angezeigt

       Die Regel dafür ist, wenn die letzten Zeichen im Präfix <MOD-DIA> oder Ø enthalten wird ein Ø Zeichen

       in der Passungstabelle gesetzt

Version 8.2.0.0

       Einstellung der Region und Sprache sollten keinen Einfluss mehr auf die Dezimalrechnung haben
       Log-Datei wird im Verzeichnis C:\Users\Benutzername\AppData\Roaming\nahe\Passungstabelle gespeichert
       damit sollte auch das Problem mit fehlenden Schreibrechten behoben sein
       zusätzlich wird geprüft ob in dem Verzeichnis der Log-Datei eine Datei erstellt werden kann

       falls das nicht möglich ist, wird das Makro trotzdem ordnungsgemäß ausgeführt

Version 8.1.0.3

       Fehler behoben:
       Berechnung der automatischen Spaltenbreite verbessert

       Berechnung der Zeilenhöhe
       Bohrungstabelle wird richtig auch ausgewertet wenn Bezeichnungen komprimiert sind

Sichtbarkeit von Bemaßungen wird berücksichtigt

Version 8.1.0.2

       Fehler behoben:
       wenn Kopfzeile unten, wurde Tabelle falsch formatiert
       Bug von SolidWorks 2019 setzen der Breite der Rasterlinienbreite behoben,
       durch Verwendung von ModelDoc2.Extension.InsertGeneralTableAnnotation an Stelle von DrawingDoc.InsertTableAnnotation

       siehe dazu auch SPR 1118249

Version 8.1.0.1

       Zusätzliche Spalten für die Anzahl der Maß/Passungskombinationen und der Zeichenblattzone

       Fehlerbereinigung bei automatischer Bestimmung der Spaltenbreite

       Funktionen zum Setzen von Standardwerten, wenn Felder in der Setup.XML Datei nicht gefunden werden, hinzugefügt

       (vor allem in Hinblick auf zukünftige Erweiterungen)
       Log-Datei wird nicht mehr in das Installationsverzeichnis geschrieben, sonder in das Verzeichnis
       C:\ProgramData\nahe\Passungstabelle\Passungstabelle.log
       Installation erweitert, damit Setup.XML Datei in ein eigenes Verzeichnis installiert werden kann
       zusätzliche Fehlerabfragen beim Lesen und Speichern der Setup.XML Datei
